## 2026-05-16 - Dynamic ARIA Labels in React\n**Learning:** Implementing accessibility for interactive stateful components requires dynamic `aria-label` attributes that accurately reflect the current state (e.g., toggling between 'Save article' and 'Remove from saved' based on a boolean value or array inclusion).\n**Action:** Always verify if an interactive element has multiple states. If so, ensure its `aria-label` updates dynamically to provide accurate context to screen readers, rather than hardcoding a single, static action description.
## 2026-05-19 - Focus Management for Custom App Shell Navigation\n**Learning:** Implementing custom navigation rails (like `RailItem`) and top-bar actions (dark mode, notifications) requires explicit focus management using utilities like `focus-visible`, as standard Tailwind button resets hide focus outlines, making keyboard navigation difficult.\n**Action:** Always append `focus-visible:ring-2 focus-visible:ring-accent focus-visible:outline-none` and `aria-current='page'` (where applicable) to custom interactive components to ensure keyboard users have a clear indication of their location in the app shell.
