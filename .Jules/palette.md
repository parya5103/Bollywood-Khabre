## 2026-05-16 - Dynamic ARIA Labels in React\n**Learning:** Implementing accessibility for interactive stateful components requires dynamic `aria-label` attributes that accurately reflect the current state (e.g., toggling between 'Save article' and 'Remove from saved' based on a boolean value or array inclusion).\n**Action:** Always verify if an interactive element has multiple states. If so, ensure its `aria-label` updates dynamically to provide accurate context to screen readers, rather than hardcoding a single, static action description.

## 2024-05-18 - Properly Connecting Labels and Inputs
**Learning:** Adding `htmlFor` and `id` links `<label>`s to form inputs, not only providing vital semantic meaning to screen readers, but also creating a larger hit area for users employing a pointing device, which is a surprisingly effective quality of life update. Additionally, components like search or newsletter bars often lack visual labels; adding `aria-label` guarantees non-visual users don't miss context.
**Action:** Always link a visual `<label>` with its `<input>` or `<textarea>` (e.g. `htmlFor`/`id`), and add `cursor-pointer` to visual labels to indicate interactivity. Use `aria-label` whenever the design pattern omits an explicit text label.
