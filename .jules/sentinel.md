## 2024-05-17 - Fix Hardcoded Fallback Credentials and Secrets
**Vulnerability:** The admin login route and authentication middleware were falling back to hardcoded secrets (`admin`, `cinepulse2026`, `super_secret_jwt_key_2026`) when environment variables were absent.
**Learning:** This "fail-open" pattern allows attackers to potentially bypass authentication if the production server environment is misconfigured or fails to load `.env`. Hardcoded fallbacks are extremely dangerous and render environment isolation moot.
**Prevention:** Implement "fail-secure" practices. Check for the presence of required environment variables early (e.g., during startup or at the start of a route handler) and fail loudly (return a 500 error or crash the app) if they are missing. Never hardcode credentials in source code.
## 2024-05-18 - ReDoS and NoSQL Injection via Unescaped RegExp
**Vulnerability:** User-provided query parameters (`q` and `category`) were passed directly into the `RegExp` constructor in `backend/src/routes/newsRoutes.js` without string casting or escaping, opening the system up to ReDoS attacks (e.g. `?q=a.*b.*c.*`) and potential NoSQL object injections.
**Learning:** MongoDB string queries utilizing the `RegExp` constructor are vulnerable to denial of service if complex user regex inputs are parsed by the database regex engine. Passing objects instead of strings via query parameters can also break expected logic.
**Prevention:** Always validate that Express query parameters intended for string logic are explicitly strings (`typeof q === 'string'`) and reliably escape any regex special characters via a function like `str.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')` before creating new `RegExp` objects.
